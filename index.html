Mysql 锁

# Mysql 锁 # 定义 锁是多线程，在并发访问共享资源时的协调机制，保证数据并发访问的一致性，有效性。 锁冲突会影响数据库并发性能。 innodb 默认行级锁，行锁是通过给索引上的索引项加锁来实现的。只有通过索引检索数据，才能使用行锁，否则将使用表锁。 # 锁的粒度 # 表锁 开销小，加锁快，不会出现死锁；锁的粒度大，发生锁冲突的几率高，并发性能低 # 行锁 开销大，加锁慢，会出现死锁；锁的粒度小，发生锁冲突的几率低，并发性能相对较高 行锁是基于索引来实现的，只有通过索引字段作为检索条件时，使用行级锁，否则使用表锁 # 锁的类型 #...
