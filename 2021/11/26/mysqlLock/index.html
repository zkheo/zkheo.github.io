# Mysql 锁

# 定义

锁是多线程，在并发访问共享资源时的协调机制，保证数据并发访问的一致性，有效性。 锁冲突会影响数据库并发性能。

innodb 默认行级锁，行锁是通过给索引上的索引项加锁来实现的。只有通过索引检索数据，才能使用行锁，否则将使用表锁。

# 锁的粒度

# 表锁

  • 开销小，加锁快，不会出现死锁；锁的粒度大，发生锁冲突的几率高，并发性能低

# 行锁

  • 开销大，加锁慢，会出现死锁；锁的粒度小，发生锁冲突的几率低，并发性能相对较高
  • 行锁是基于索引来实现的，只有通过索引字段作为检索条件时，使用行级锁，否则使用表锁

# 锁的类型

# 共享锁

  • 也称为读锁，共享读，当一个事务对数据添加共享锁后，其他事务也可以对该数据添加共享锁，但不能添加排他锁

# 排它锁

  • 也称为写锁，当一个事务对数据添加排他锁后，其他事务，不能对该数据添加任何锁

# 意向锁

  • （IS）意向共享锁，当一个事务对数据申请共享锁时，会先申请意向共享锁
  • （IX）意向排他锁，当一个事务对数据申请排他锁时，会先申请意向排他锁

# 意向锁的意义

假设，事务 A 获取了某一行的排它锁，尚未提交，此时事务 B 想要获取表锁时，必须要确认表的每一行都不存在排他锁，很明显效率会很低，引入意向锁之后，只需要判断表级意向锁即可，效率就会大为改善

# 间隙锁

当我们用范围条件检索数据而不是相等条件检索数据，并请求共享或排他锁时，InnoDB 会给符合范围条件的已有数据记录的索引项加锁；对于键值在条件范围内但并不存在 的记录，叫做 “间隙（GAP)”。
