# Mysql MVCC

# 概要

MVCC ，全称 Multi-Version Concurrency Control ，即多版本并发控制。在发生读写冲突时，能做到不加锁，非阻塞并发读，简单来说就是为了解决 读写冲突 ，提高数据库并发性能

# 当前读、快照读

在学习 MVCC 多版本并发控制之前，需要了解一下，什么是 MySQL InnoDB 下的 当前读 和 快照读

# 当前读

  • select lock in share (共享锁), select for update; update; insert; delete (排他锁)，这些操作都是当前读，读取时还要保证其他并发事务不能修改当前记录，会对读取的记录进行加锁

# 快照读

  • 普通的 select 操作就是快照读，不加锁的非阻塞读。快照读的前提是隔离级别不是串行级别，串行级别下的快照读会退化成当前读；之所以出现快照读的情况，是基于提高并发性能的考虑，快照读的实现是基于多版本并发控制，即 MVCC, 可以认为 MVCC 是行锁的一个变种，但它在很多情况下，避免了加锁操作，降低了开销；既然是基于多版本，即快照读可能读到的并不一定是数据的最新版本，而有可能是之前的历史版本

# MVCC 实现原理

通过 3个隐式字段 ， undo日志 ， Read View 来实现的

# 隐式字段

  • DB_TRX_ID：最近修改 ( 修改/插入 ) 事务 ID：记录创建这条记录 / 最后一次修改该记录的事务 ID
  • DB_ROLL_PTR：回滚指针，指向这条记录的上一个版本
  • DB_ROW_ID：自增 ID（隐藏主键），如果数据表没有主键，InnoDB 会自动以 DB_ROW_ID 产生一个聚簇索引

# undo log

  • 当一个数据被修改时，会对数据添加 排他锁
  • 然后把该行数据拷贝到 undo log 中，作为旧记录，既在 undo log 中有当前行的拷贝副本
  • 然后修改当前当前数据，并且将当前数据的隐式字段事务 ID 修改为当前事务的 ID，然后将回滚指针指向 undo log 的副本记录，表示当前记录的上一个版本
  • 当数据再次被修改时，会将最新的旧数据插入到链表的头部

# Read View 读视图

Read View 就是事务进行 快照读 操作的时候生产的 读视图 ，记录并维护系统当前活跃事务的 ID，通过 读视图 来判断当前事务能够读取到哪个版本的数据，可能是最新的数据，也有可能是 undo log 中的历史版本
